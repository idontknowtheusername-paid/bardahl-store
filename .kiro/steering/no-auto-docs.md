# Règle : Interdiction de création automatique de documentation

**RÈGLE STRICTE** : Ne créez JAMAIS de fichiers de documentation automatiquement, sauf si l'utilisateur le demande explicitement.

## Fichiers concernés

- Fichiers markdown de résumé (SUMMARY.md, RECAP.md, etc.)
- Fichiers de documentation technique non demandés
- Fichiers README supplémentaires
- Fichiers de notes ou de logs
- Tout fichier .md qui n'est pas directement requis pour le fonctionnement du projet

## Comportement attendu

- ✅ Fournir un résumé verbal concis à la fin du travail
- ✅ Créer de la documentation uniquement sur demande explicite
- ❌ Ne PAS créer de fichiers markdown pour documenter votre processus
- ❌ Ne PAS créer de fichiers de résumé automatiquement
- ❌ Ne PAS créer de documentation "utile" non sollicitée

## Résumés

Les résumés doivent être :
- Verbaux uniquement (dans la réponse)
- Très concis (2-3 phrases maximum)
- Sans listes à puces inutiles
- Focalisés sur le résultat final

Cette règle s'applique à toutes les interactions, sauf demande contraire explicite de l'utilisateur.
